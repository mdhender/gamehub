# Galaxy Generation

Generation builds a game's galaxy deterministically from a seed string. The pipeline runs in four sequential steps, each continuing the PRNG state left by the previous step. Every step is recorded in the `generation_steps` table, making the full sequence auditable and replayable.

## PRNG

All randomness flows through `GameRng`, which wraps PHP's `Xoshiro256StarStar` engine. The seed string is hashed with SHA-256 to produce the 256-bit engine state. Between steps, the engine state is serialized to a string and stored on the `games` row as `prng_state`. Each step picks up exactly where the previous one left off, so the same seed always produces an identical galaxy.

The primitive operations used throughout generation are:

- `int(min, max)` — uniform random integer, inclusive on both ends
- `rollDice(n, sides)` — sum of `n` independent `int(1, sides)` rolls

---

## Step 1 — Stars

**Class:** `StarGenerator`
**Consumes:** `prng_seed` (fresh seed, not continuing state)
**Produces:** 100 `Star` records

The cluster is a 31×31×31 coordinate cube. Each axis runs 0–30, giving 29,791 possible positions. 100 stars are placed by rolling three independent coordinates per star.

```
for i = 0 to 99:
    x = int(0, 30)
    y = int(0, 30)
    z = int(0, 30)
```

Stars that land on the same coordinates form a **system group**. Each star in a group receives a 1-based `sequence` number assigned in generation order. The unique key for a star is `(game_id, x, y, z, sequence)`.

## Step 2 — Planets

**Class:** `PlanetGenerator`
**Consumes:** PRNG state from step 1
**Produces:** 1–10 `Planet` records per star

Planet generation runs two sub-phases for each star in database ID order: orbit assignment, then habitability rolling.

### Orbit Assignment

Each star has 10 orbital slots (orbits 1–10). Each slot is rolled independently:

| Roll (1–100) | Result |
|---|---|
| 1–29 | Terrestrial (29%) |
| 30–34 | Asteroid Belt (5%) |
| 35–41 | Gas Giant (7%) |
| 42–100 | Empty (59%) |

**Fallback:** If all 10 slots roll empty, the star receives a fixed default layout: 5 terrestrials, 2 asteroid belts, 3 gas giants.

**Caps** are then applied to prevent extreme compositions:
1. If gas giants > 3, convert the excess to asteroid belts starting from orbit 1.
2. If asteroid belts > 2, convert the excess to terrestrials starting from orbit 1.

**Sorting:** The set of occupied orbit positions is collected in order. Planet types are then sorted — Terrestrial < Asteroid Belt < Gas Giant — and redistributed back across those positions. This means terrestrials always occupy the lowest occupied orbits, gas giants the highest.

### Habitability

Rolled immediately after each planet's type is determined, using the stored orbit number (1–10).

**Asteroid Belt:** Always 0. No RNG roll consumed.

**Gas Giant** — `int(0, max)` by orbit:

| Orbit | Range |
|---|---|
| 1 | 0–2 |
| 2 | 0–4 |
| 3 | 0–7 |
| 4 | 0–10 |
| 5 | 0–13 |
| 6 | 0–16 |
| 7 | 0–19 |
| 8 | 0–13 |
| 9 | 0–7 |
| 10 | 0–1 |

**Terrestrial** — dice expression by orbit:

| Orbit | Expression | Range |
|---|---|---|
| 1 | 1d3 − 1 | 0–2 |
| 2 | 2d3 − 2 | 0–4 |
| 3 | 3d4 − 3 | 0–9 |
| 4 | 2d10 | 2–20 |
| 5 | 2d12 + 1 | 3–25 |
| 6 | 2d10 | 2–20 |
| 7 | 3d4 − 3 | 0–9 |
| 8 | 2d3 − 2 | 0–4 |
| 9 | 1d3 − 1 | 0–2 |
| 10 | 1d2 − 1 | 0–1 |

Habitability can be 0 for any planet type. A terrestrial with habitability 0 is a barren world exploitable by advanced technology.

## Step 3 — Deposits

**Class:** `DepositGenerator`
**Consumes:** PRNG state from step 2
**Produces:** 0–34 `Deposit` records per planet

Planets are processed in database ID order. For each planet:

```
depositCount = int(0, 34)
```

Each deposit is then generated by a table specific to the planet's type.

### Asteroid Belt Deposits

| Roll (1–100) | Resource | Quantity | Yield |
|---|---|---|---|
| 1 | Gold | 100,000–5,000,000 | 1d3 |
| 2–10 | Fuel | 1,000,000–99,000,000 | 3d6 − 2 |
| 11–100 | Metallics | 1,000,000–99,000,000 | 3d10 − 2 |

### Gas Giant Deposits

| Roll (1–100) | Resource | Quantity | Yield |
|---|---|---|---|
| 1–15 | Fuel | 1,000,000–99,000,000 | 10d4 − 2 |
| 16–40 | Metallics | 1,000,000–99,000,000 | 10d6 |
| 41–100 | Non-Metallics | 1,000,000–99,000,000 | 10d6 |

### Terrestrial Deposits

Yield formulas branch on whether the planet's habitability is 0 (barren) or greater (inhabited).

| Roll (1–100) | Resource | Quantity | Yield (barren) | Yield (inhabited) |
|---|---|---|---|---|
| 1 | Gold | 100,000–1,000,000 | 1d3 | 3d4 − 3 |
| 2–15 | Fuel | 1,000,000–99,000,000 | 10d4 − 2 | 10d8 |
| 16–45 | Metallics | 1,000,000–99,000,000 | 10d6 | 10d8 |
| 46–100 | Non-Metallics | 1,000,000–99,000,000 | 10d6 | 10d8 |

Gold on an inhabited world rolls 3d4 − 3, which has a minimum of 0. A zero-yield gold deposit is recorded as exhausted and may be exploitable by later technology levels.

---

## Generation Steps

Each step writes a `GenerationStep` record with:

| Field | Description |
|---|---|
| `step` | `Stars`, `Planets`, or `Deposits` |
| `sequence` | Monotonically increasing integer across all steps for the game |
| `input_state` | Serialized PRNG state at the start of the step |
| `output_state` | Serialized PRNG state at the end of the step |

Storing both states allows any step to be replayed in isolation or the full sequence to be verified.
